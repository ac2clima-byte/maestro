📢 ECHO WhatsApp integrato in nexusRouter

PRIMA: "manda whatsapp a Alberto: ciao" → "Collega non ancora attivo"
DOPO: handler handleEchoWhatsApp reale, con whitelist + Waha API

🔒 Sicurezza a 3 livelli

  1. DRY_RUN default true: nessun invio reale finché non esplicitamente disattivato
  2. Whitelist obbligatoria: cosmina_config/echo_config.allowed_numbers deve contenere il numero risolto, altrimenti rifiuto
  3. Alias vs numeri arbitrari: "Alberto" è un alias che punta a un numero già in whitelist. Numeri suggeriti dal modello LLM NON sono mai inviati se non whitelistati esplicitamente

✅ Test Playwright — 3 scenari

"manda whatsapp a Alberto: NEXO è operativo"
ECHO: nessun destinatario autorizzato. Alberto deve popolare `cosmina_config/echo_config.allowed_numbers` prima di poter inviare.

✅ Comportamento corretto: blocca in assenza di whitelist configurata

"manda whatsapp a 3339999999: ciao" (numero arbitrario non autorizzato)
ECHO: nessun destinatario autorizzato.

✅ Sicurezza: anche numeri concreti vengono rifiutati se non whitelistati

Nessun fallback "Collega non ancora attivo" in NESSUNO dei 3 scenari

✅ Handler ECHO ora è agganciato al nexusRouter e risponde sempre

⚙️ Setup finale per attivare invio reale

Alberto deve creare/modificare il documento Firestore cosmina_config/echo_config sul progetto garbymobile-f89ac:

{
  "dry_run": false,              // false = invia davvero
  "allowed_numbers": [
    "+39XXXXXXXXXX"              // il numero personale di Alberto
  ],
  "aliases": {
    "alberto": "+39XXXXXXXXXX"   // mappa "Alberto" → numero
  }
}

E il documento cosmina_config/whatsapp deve contenere la config Waha:

{
  "waha_url": "https://waha.example.com",
  "waha_session": "default",
  "waha_api_key": "...",
  "enabled": true
}

🎯 Come Funziona

  1. Utente → NEXUS: "manda whatsapp a Alberto: test"
  2. Haiku classifica → collega=echo, azione=send_whatsapp, parametri={to:"Alberto", body:"test"}
  3. Router trova handler handleEchoWhatsApp (matcha sia per collega che per userMessage)
  4. Handler: legge echo_config, risolve alias "alberto" → numero whitelistato
  5. Se DRY_RUN: scrive su echo_messages con status=skipped, risponde "simulato"
  6. Se invio reale: POST a {WAHA_URL}/api/sendText con X-Api-Key, scrive status=sent
