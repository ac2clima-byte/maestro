Componenti modificati

handlers/nexus.jsCounter atomico DEV-NNN su nexo_config/nexo_counters.devRequest; doc primario su nexo_dev_requests/{DEV-NNN}; coda su nexo_dev_requests_queue/{DEV-NNN} con queueStatus=pending; risposta naturale "Ho registrato la richiesta DEV-NNN: «...». Claude Code la sta analizzando, ti aggiorno quando l'analisi è pronta."
scripts/dev_request_poller.mjsLoop 15s: legge queue pending, scrive tasks/dev-request-DEV-NNN.md, fa commit/pull --rebase/push, marca queueStatus=pushed e aggiorna nexo_dev_requests.status=analyzing.
maestro.mjsgetPendingTasks() ora ritorna oggetti {id, kind, devId}. I file dev-request-*.md sono trattati come kind=dev-request: prompt speciale che impone analisi (no implementazione) e output in tasks/dev-analysis-DEV-NNN.md. Il task è "completo" quando il file analisi esiste in repo.
start-maestro.sh / stop-maestro.shAvviano/fermano anche dev_request_poller.mjs. Log in logs/dev-poller.log.
firestore.rulesAggiunta regola per nexo_dev_requests_queue (read pubblica, write Admin only).

Test end-to-end (quando MAESTRO è riavviato)

  1. ./stop-maestro.sh && ./start-maestro.sh
  2. Apri https://nexo-hub-15f2d.web.app e in NEXUS Chat scrivi: "il report mensile non parsa i mesi in italiano, vorrei che li mostri"
  3. Risposta NEXUS: "Ho registrato la richiesta DEV-001: «...». Claude Code la sta analizzando, ti aggiorno quando l'analisi è pronta."
  4. Entro 30s: il file tasks/dev-request-DEV-001.md appare su GitHub.
  5. MAESTRO lo intercetta, manda a Claude Code con prompt analisi (no implementazione).
  6. Claude Code scrive tasks/dev-analysis-DEV-001.md e pusha.
  7. Claude Chat (sessione successiva) fa git pull && ls tasks/dev-* e discute con Alberto.
