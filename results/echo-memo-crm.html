📞 ECHO — Numeri da MEMO/CRM (no whitelist statica)

Prima: whitelist statica in echo_config.allowed_numbers — fragile, PII duplicata.
Dopo: i numeri sono dove già sono — in COSMINA (tecnici_acg, contatti_clienti, numeri_acg).

🔍 Pipeline di risoluzione destinatario

Quando l'utente scrive "manda WA a [nome]", il router cerca in ordine:

PrioritàFonteMatchingCampo numero
1cosmina_config/tecnici_acg.tecnici[]nome, cognome, nome_completo, report_aliastelefono / whatsapp / cellulare / tel
2cosmina_config/numeri_acg (opzionale)chiave → valore stringa numerovalore
3cosmina_contatti_clientinome_completo / nome+cognometelefono_normalizzato → telefono

🛡️ Safety model

  • DRY_RUN default true (flag in cosmina_config/echo_config.dry_run)
  • Niente numeri grezzi dal prompt: "manda a 3331234567" → rifiuto immediato
  • Niente whitelist statica: solo contatti del CRM ricevono
  • Numero mascherato in output: +393***4567 invece del numero completo
  • Alberto aggiunge numeri tecnici in cosmina_config/tecnici_acg.tecnici[].telefono o in cosmina_config/numeri_acg = {"malvicino": "+39..."}

✅ Test Playwright — 4 scenari

"manda whatsapp a Malvicino: domani intervento Kristal ore 14"
❓ Non trovo il numero di "Malvicino" nel CRM. Verifica in COSMINA che il contatto esista. Per tecnici senza contatto CRM, Alberto può aggiungere il numero in cosmina_config/tecnici_acg o cosmina_config/numeri_acg.

✅ Corretto: Malvicino è in tecnici_acg ma senza campo telefono → guida l'utente verso la config.

"manda whatsapp a Alberto: NEXO funziona"
⚠️ Trovo più contatti per "Alberto" nel CRM: · Alberto Armano - CONDOMINIO MAGGIOCIONDOLO · Alberto Perduca Sii più specifico (nome + cognome).

✅ Ambiguità correttamente segnalata. Alberto il proprietario non è tra i risultati perché non è in contatti_clienti (è solo in tecnici_acg senza telefono).

"manda whatsapp a 3339999999: spam"
🚫 ECHO rifiuta numeri grezzi dal prompt per sicurezza. Indica il destinatario per NOME (tecnico, cliente, condominio).

✅ Safety critica: un LLM compromesso non può spammare numeri arbitrari.

"manda whatsapp a Pluto Topolino: prova"
❓ Non trovo il numero di "Pluto Topolino" nel CRM.

✅ Nome inesistente rifiutato correttamente.

⚙️ Setup per attivare invio reale

Alberto popola in Firestore (progetto garbymobile-f89ac):

1. Per tecnici senza contatto CRM (es. Malvicino), aggiungi il campo telefono dentro tecnici_acg:

cosmina_config/tecnici_acg = {
  tecnici: [
    { id: 2, nome: "ROBERTO", cognome: "MALVICINO", telefono: "+39XXXXXXXXXX", attivo: true },
    ...
  ]
}

Oppure (più semplice) crea una mappa dedicata:

cosmina_config/numeri_acg = {
  alberto: "+39XXXXXXXXXX",
  malvicino: "+39XXXXXXXXXX",
  dellafiore: "+39XXXXXXXXXX"
}

2. Attiva invio reale (dopo aver testato in DRY_RUN):

cosmina_config/echo_config = { dry_run: false }

3. Config Waha (URL + API key):

cosmina_config/whatsapp = {
  waha_url: "https://...",
  waha_session: "default",
  waha_api_key: "...",
  enabled: true
}
