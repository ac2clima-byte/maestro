Cosa è stato sistemato

NEXUS scriveva in nexo_dev_requests ma il file md non veniva creato perché:

  • Il poller esterno scripts/dev_request_poller.mjs legge solo nexo_dev_requests_queue (collection nuova introdotta nella Fase 0).
  • Le richieste pre-Fase 0 finite in nexo_dev_requests con auto-id (es. trzc6usXBnlyv7SGPu04) restavano orfane.

Aggiunta pollDevRequests() direttamente in maestro.mjs che legge la collection canonica nexo_dev_requests:

  1. Query where status == "pending" limit 10
  2. Per ogni doc: scrive tasks/dev-request-{id-sanificato}.md
  3. Aggiorna il doc Firestore con status="file_creato" e taskFile
  4. Commit + pull --rebase + push

Eseguito una volta in modalità one-shot per recuperare il backlog (1 documento storico).
