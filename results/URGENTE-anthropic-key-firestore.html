🚨 VULNERABILITÀ CRITICA: Anthropic API Key in Firestore

Durante la scansione MEMO di garbymobile-f89ac Firestore è stata rilevata una Anthropic API Key in chiaro nella collection cosmina_config (campo anthropic_api_key). GitHub Push Protection ha bloccato il primo tentativo di commit perché ha scansionato la key.

🔍 Dettagli

  • Progetto Firebase: garbymobile-f89ac
  • Collection: cosmina_config
  • Campo: anthropic_api_key (plaintext)
  • Rilevato da: MEMO scan 2026-04-22 + GitHub Push Protection
  • Tipo: Anthropic API Key production

🎯 Impatto

  • Chiunque abbia read su cosmina_config può leggere la chiave (dipende dalle Firestore rules)
  • Se le rules permettono lettura ampia → chiave visibile a qualsiasi utente autenticato
  • La chiave è usata per chiamate a Claude API → un attaccante può consumare budget Anthropic
  • Rischio di esfiltrazione: la chiave può essere clonata e usata da terzi

✅ Azioni immediate richieste

Step 1 — Rotare la chiave
  1. Accedi a console.anthropic.com
  2. Revoca la chiave attualmente in cosmina_config.anthropic_api_key
  3. Crea una nuova chiave con nome descrittivo (es. "ACG Suite — COSMINA 2026-04-22")
Step 2 — Spostare la chiave in Secret Manager 
firebase functions:secrets:set ANTHROPIC_API_KEY --project garbymobile-f89ac
# Inserisci la NUOVA chiave quando richiesto
Poi nel codice Cloud Function COSMINA: 
// Sostituisci:
const key = (await db.collection('cosmina_config').doc('X').get()).data().anthropic_api_key;

// Con:
import { defineSecret } from "firebase-functions/params";
const ANTHROPIC_API_KEY = defineSecret("ANTHROPIC_API_KEY");
// ... nell'onRequest: secrets: [ANTHROPIC_API_KEY]
const key = ANTHROPIC_API_KEY.value();
Step 3 — Rimuovere la chiave dal documento Firestore

Dopo aver messo la chiave in Secret Manager e deployato le functions che la usano:

# Da Python con firebase_admin:
db.collection('cosmina_config').document('<doc_id>').update({
    'anthropic_api_key': firestore.DELETE_FIELD
})
Step 4 — Verificare rules Firestore

Controlla che cosmina_config abbia read: if false (o almeno auth required):

match /cosmina_config/{docId} {
  allow read: if request.auth != null && ruolo admin;
  allow write: if false; // solo Admin SDK
}
Step 5 — Audit chi ha letto la chiave

Controlla audit_log per endpoint che leggono cosmina_config:

db.collection('audit_log').where('endpoint', '>=', 'GET /api/config').limit(100).stream()

Fare un audit sistematico di tutti i campi Firestore che contengono secret e spostarli in:
