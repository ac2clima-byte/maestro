Logica del contatore

const STATUS_REPORT_EVERY = 8;            // 8 cicli × 15s = 120s
let ciclo = 0;
writeAndPushStatusReport();               // primo report allo startup
while (true) {
  await pull();
  ...processa task...
  ciclo++;
  if (ciclo % STATUS_REPORT_EVERY === 0) {
    writeAndPushStatusReport();           // ~ogni 2 minuti
  }
  await sleep(POLL_INTERVAL);
}

Cambio rispetto al commit precedente (40cba93)

Nel commit precedente STATUS.md veniva scritto al cambio fase, con dedup per fasi noisy. Questo task chiede una soluzione diversa, più semplice:

  • Niente dedup per fase: il push avviene sempre ogni 8 cicli del main loop.
  • Aggiungo task pending + ultimi 5 commit (utile per Claude Chat senza altri tool).
  • Lo stato viene letto via curl dalla stessa Cloud Function della dashboard, così STATUS.md e dashboard restano coerenti.
